# 版本更新紀錄 (v6_Quiz_Fixes)

## 變更概述
本次更新主要為了解決「互動式遊戲問卷 (Interactive Quiz)」 在行動裝置上發生的選項遮蔽、畫面截斷，以及表單防呆與系統自動推算的優化。這些調整確保了網頁在不同型號手機上的滑動體驗，同時也替使用者省去記憶專業數據（如 BMR 與體脂肪）的麻煩。

## 具體變更項目
1. **優化選項頁面顯示防截斷 (`quiz.html`, `quiz.css`)**：
   - 解決因為 `position: absolute` 與固定 `align-items: center` 造成的畫面溢位。修改為新增 `.scrollable-card` 樣式，當內部元素過多時，卡片會切換為可上下順暢滑動 (`overflow-y: auto`)，並保留足夠的上下留白。
2. **防呆按鈕設計 - 顏色深淺 (`quiz.css`, `quiz.js`)**：
   - 取消原本可能只依賴警告窗的防呆機制。為「下一步」按鈕實裝 `.btn-disabled` 灰色未啟用狀態。
   - 在所有的必填欄位加入 JS `checkValidity` 監聽，客戶只有正確填寫數字後，按鈕才會「點亮」成為可進入下一關的彩色按鈕。
3. **導覽列全局統一 (`所有 html`)**：
   - 將「互動測驗 Game」的分頁連結，確實佈署到 `contact.html`、`core-value.html`、`pain-points.html`、`solution.html` 等每一頁的全域 Nav 導覽列上，避免使用者在站內迷路。
4. **系統智能推算 - 免除體脂與 BMR 填寫 (`quiz.js`, `quiz.html`)**：
   - 把原本的「填寫體脂率」與「填寫 BMR」的闖關卡片**移除**，改由系統幕後自動計算。
   - **BMR (基礎代謝)**：套用全球最通用的 **Mifflin-St Jeor 公定方程式** (以性別、實足年齡、身高、體重交叉計算)。
   - **體脂率 (BFP)**：套用以 BMI 為基礎的標準體脂率推演公式系統：`1.2 × BMI + 0.23 × 年齡 - 10.8 × 性別參數 - 5.4`。

## 老師的設計巧思
- 將填寫題目由 7 題精簡到 5 題！體脂跟 BMR 對普通人來說門檻太高了，如果不拿掉客戶很容易在這裡卡關放棄。
- 取代原本在 `alert()` 跳出紅色警告文字的方法，改用灰階鎖定按鈕的概念，這種**「無聲的視覺引導」**是高級 APP 介面中最常用的 UX (使用者體驗) 技巧。
