# 版本更新紀錄 (v7_Quiz_Layout)

## 變更概述
在收到使用者回饋指出「內容框架被截斷、不想要強制出現捲軸、且 RWD 希望文字自然順展」的嚴格體驗要求後，本次 v7 版本實行了一次前端結構的「去絕對定位手術」。

## 具體變更任務清單 (quiz.css / quiz.html)
1. **捨棄浮動高度限制 (`quiz.css`)**：
   - 撤銷之前作為特效代價而套用在 `.quiz-card` 的 `position: absolute`，全面改為 `display: none` 作為起手式。這會釋放卡片的自然流（Natural Flow）。
   - **為什麼這樣做？** HTML 原生的優勢就是「內容如果有一萬字，它的外皮箱子就會跟著被長大到可以包住一萬字」。但如果子元素是 Absolute（絕對定位）就像幽靈，會使得箱子根本感應不到裡面的字，就會發生超出邊界、必須得自己加 `min-height` 的窘境。現在，這困擾正式解除了！只要該 `.quiz-card.active` 出現的時候，它就會瞬間跟背景的 `quiz-container` 箱子同化，幫它撐開高度！
2. **清除無用的 Scrollable 防護網**：
   - 拔除 v6 加入的防溢出樣式 `.scrollable-card`。既然外框現在已經「完全彈性長大」，就算客戶用最小台的行動裝置填寫第五題的進階數據，版面也是跟隨著他們上下划手機螢幕，而不需要在卡片「裡面」又生出一個怪異的捲動軸。
   - `quiz.html` 中相關的 wrapper class `.scroll-content` 同樣被剃除乾淨。
3. **清除行動裝置的不規則預留高度 (`quiz.css`)**：
   - 先前在 `@media` 查詢的 `380px` 與 `768px` 手機斷點中，所強制綁定的 `min-height: 520px / 580px` 全部下架。框架正式宣告進入**完全智慧化隨文字內容變長**的完美 RWD。
